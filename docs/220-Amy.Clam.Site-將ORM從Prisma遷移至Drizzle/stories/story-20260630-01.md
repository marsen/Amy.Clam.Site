# 開發者 → 透過 Drizzle 完成所有資料庫操作，現有功能不受影響

**日期**：2026-06-30
**想法**：#220

## 互動關係

**背景**：Amy.Clam.Site 目前以 Prisma 存取資料庫，違反團隊統一使用 Drizzle 的開發原則。

**期望結果**：

- 所有原本透過 Prisma 完成的資料庫操作（商品查詢、訂單管理、報表等）改由 Drizzle 處理
- 使用者看到的功能行為與遷移前完全一致
- 部署流程不再需要 Prisma code generation 步驟
- 專案中不再存在 Prisma 相關依賴

**邊界條件**：

- 資料庫結構（schema）不得改變，現有資料須完整保留
- 遷移期間如需停機，需事先規劃
